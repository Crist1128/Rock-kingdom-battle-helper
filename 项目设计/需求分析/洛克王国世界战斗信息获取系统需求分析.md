# 洛克王国世界战斗信息获取系统需求分析

## 一、项目概述

开发一个用于**洛克王国世界PVP对战**的战斗信息获取和计算工具，帮助玩家在战斗中：

- **计算我方技能对敌方精灵的伤害（百分比生命）**
- **计算敌方技能对我方精灵的伤害（精确整数值）**
- **实时获取敌方精灵属性**（培养配置）
- **基于伤害判断做出决策**

**策略判断由玩家自行完成**，程序仅提供数据与计算支持。

**核心目标：**

1. 反推敌方精灵“性格“与”种族值（生命值、物攻、物防、魔攻、魔防、速度）“。
2. 计算我方精灵技能对敌方精灵的伤害。
3. 计算敌方精灵技能对我方精灵的伤害。
4. 对未知敌方精灵或技能使用数据库可能配置的属性/技能组进行估算。
5. 通过图像识别获取战斗信息，包括**Buff/Debuff状态、敌方对位精灵、技能伤害数值、**~~技能克制关系（不需要通过识别获取，只需要调动属性）~~，**可选**手动输入。
6. 所有计算结果在**独立程序页面**显示。
7. 等级固定为60级；技能直接调用技能库。
8. 暂未实现对场下精灵计算，可能考虑未来更新（点击更换键选择精灵，点击识别，更新计算）

## 二、功能需求

### 1. 数据输入

- **己方精灵信息**（玩家提前输入）：
    - 种族值：生命、物攻、物防、魔攻、魔防、速度（精确值）
    - 技能信息：名称、调用技能库
    - 实时Buff/Debuff状态（包括冻结）
- **敌方精灵信息**：
    - 通过图像识别获取精灵头像、技能图标、Buff/Debuff状态
    - 图像识别有固定自动识别。
    - 应有手动单次识别按钮，目的：buff堆积时手动打开列表补充、对方使用招式时手动点开精灵信息进行识别补充
    - 图像识别失败与否都允许手动输入补充，手动补充优先级更高
    - 手动补充可以考虑关键字索引，提高输入效率
- **数据库**：
    - 全体精灵属性配置方案（性格、个体值、基础属性）
    - 技能组（基础威力，属性类别，技能特性，基于技能特性的威力改变，消耗能量）

### 2. 图像识别与信息获取

- OCR/图像识别捕获：
    - 敌方精灵头像识别精灵信息
    - 技能图标识别技能信息
    - 双方Buff/Debuff状态栏信息
    - ~~技能旁显示的实时伤害数值~~
    - ~~技能克制关系标识~~
- 数据缓存：
    - 敌方精灵属性、~~血量（血量实际上不需要）~~、技能信息
    - 双方现有Buff/Debuff，印记，天气（天气实现难，建议手动输入）
    - 双方后台精灵的状态（精灵信息，冻结层数，能量）
    - 我方技能信息
- 失败处理：
    - 用户手动重试识别
    - 用户手动输入补充（若手动输入，则输入信息优先级应高于识别信息）

### 3. 敌方属性推算

- 根据实际伤害数据和己方技能属性反推敌方精灵种族值
- 考虑**Buff/Debuff状态及技能克制关系**
- 多次伤害累积可提高精确度
- ~~暂不考虑暴击、随机波动（洛克没有暴击，随机数）~~
- 更新缓存表记录敌方最新属性

### 4. 技能伤害计算

### 4.1 我方技能对敌方精灵

- 局外输入：我方技能列表 + 我方精灵属性值
- 局内输入： 己方技能 + 敌方已知/估算属性值
- 运算：调用技能数据库 + Buff/Debuff状态 + 克制关系 ——进入伤害计算公式进行运算
- 输出：我方技能对每个敌方精灵造成伤害——生命百分比
- 对未知敌方精灵使用数据库常见培养精灵数值进行估算区间

### 4.2 敌方技能对我方精灵

- 局外输入：我方属性
- 局内输入：敌方技能**（已知或常见技能组**（要做实时更新，随战斗发展逐渐已知）**）** + Buff/Debuff状态
- 运算：调用技能数据库 + Buff/Debuff状态 + 克制关系 ——进入伤害计算公式进行运算
- 输出：敌方技能对每只我方精灵造成伤害——整数值

### 5. 信息展示

- **独立程序页面**显示所有计算结果：
    - 敌方精灵及其属性、Buff/Debuff状态
    - 我方技能对敌方精灵伤害
    - 敌方技能对我方精灵伤害
    - ~~技能克制关系信息~~
- 支持动态刷新，随战斗实时更新
- 表格示例：
    
    ```
    我方技能列表：
      技能X: 伤害%
      技能Y: 伤害%
      技能Q: 伤害%
      技能Z: 伤害%
    
    敌方技能：
      技能A: 伤害值
      技能B: 伤害值
    
    ```

## 三、非功能需求

1. **实时性**：信息识别、计算与展示快速响应，适用于PVP实时战斗
2. **准确性**：
    - 属性推算基于多次伤害累积，减少误差
    - 技能伤害精确到整数或百分比
    - Buff/Debuff识别准确，伤害计算输出正确
3. **易用性**：
    - 用户手动识别或输入简洁直观
    - 信息展示清晰，便于快速查看
4. **可扩展性**：
    - 更新新精灵、新技能、新Buff/Debuff
    - 支持切换时，更新计算预备入场精灵的伤害
    - 支持增加对手后台迅捷入场技能的计算

## 四、数据需求

### 1. 用户输入数据

- 己方精灵：
    - 名称（进行索引）
    - 种族值：生命、物攻、物防、魔攻、魔防、速度
    - 技能信息：名称（进行索引）

### 2. 数据库

- 精灵数据库：默认种族值、属性、技能列表
- 技能数据库：威力、属性类型、相关特性
- 图像数据库：精灵头像、技能图标、Buff/Debuff图标

### 3. 战斗实时数据

- 敌方精灵能量、技能使用、Buff/Debuff状态
- 每次伤害数值（己方造成/敌方造成）
- 技能克制关系（通过图像识别或推算）

## 五、模块设计

| 模块 | 功能 | 输入 | 输出 |
| --- | --- | --- | --- |
| 数据输入模块 | 用户输入己方精灵信息及手动补充敌方信息 | 用户表单 | 内存数据结构 |
| 图像识别模块 | OCR/图像识别获取敌方精灵、技能、Buff/Debuff、 | 屏幕截图 | 敌方精灵属性、技能、伤害、Buff/Debuff |
| 敌方属性推算模块 | 根据伤害、Buff/Debuff和克制状态反推敌方属性 | 我方技能 + 实际伤害 + Buff/Debuff | 敌方种族值估算结果 |
| 技能伤害计算模块 | 计算技能伤害 | 使用技能 +双方属性 + Buff/Debuff + 其他因素 | 伤害数值/百分比 |
| 信息展示模块 | 独立页面展示所有信息 | 技能伤害计算结果 + Buff/Debuff状态 | 用户可视表格/列表 |

---

## 六、边界条件与假设

1. 等级统一为60级
2. 技能仅包含名称、威力、属性类型，没有命中率
3. 没有暴击、随机波动
4. 技能克制/抵抗通过后台逻辑直接接入伤害计算公式
5. 敌方未知精灵或技能使用数据库默认值
6. 属性推算精度依赖多次伤害累积
7. 图像识别失败与否都需允许用户手动补充数据
8. 信息量大，必须使用独立页面展示
