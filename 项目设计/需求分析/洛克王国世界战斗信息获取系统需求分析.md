# 洛克王国世界 PVP 战斗信息获取与敌方配置推算系统需求分析 v0.2

## 一、项目概述

本项目开发一个用于 **洛克王国世界 PVP 对战** 的战斗信息获取、敌方配置推算与伤害计算工具。

工具的核心目标不是自动替玩家做决策，而是在对战过程中，基于准备阶段可见的双方六只宠物、己方已知完整配置、敌方未知配置，以及战斗中产生的伤害、技能、Buff/Debuff、天气、异常状态等信息，持续推算敌方宠物的实际配置，并实时展示双方技能伤害结果，辅助玩家进行判断。

## 二、核心需求

对战开始前，系统可以看到双方出场的六只宠物。

己方六只宠物的信息由玩家提前输入，因此己方宠物的以下信息是已知的：

- 宠物名称
- 等级，固定为 60 级
- 六维种族值
- 性格
- 培养值
- 技能组
- 实际六维属性
- 当前 Buff/Debuff、异常状态、能量、生命值等战斗状态

敌方六只宠物在准备阶段只能知道宠物种类，不能直接知道具体配置。敌方未知信息包括：

- 性格
- 培养值
- 实际六维属性
- 技能组
- 当前技能使用情况
- 后续战斗状态变化

系统通过战斗过程中双方造成的伤害，结合伤害公式，反向筛选敌方可能配置。

当我方攻击敌方造成伤害时，系统根据我方已知攻击属性、技能、Buff/Debuff、天气、异常状态等信息，反推敌方可能的生命、防御、魔防、性格、培养值等配置。

当敌方攻击我方造成伤害时，系统根据我方已知防御属性、敌方造成的准确伤害值、敌方可能技能池、Buff/Debuff、天气、异常状态等信息，反推敌方可能的物攻、魔攻、性格、培养值和使用技能。

随着战斗推进，系统不断收集伤害证据，逐步缩小敌方每只宠物的可能配置范围。

------

# 三、核心概念定义

## 1. 种族值

种族值是宠物物种本身固定的基础属性。

在准备阶段，如果敌方六只宠物的名称已经可见，则敌方宠物的种族值可以从数据库中直接读取。

因此，本系统原则上不需要“反推敌方种族值”，而是需要反推敌方在种族值基础上的实际配置。

## 2. 培养值

培养值是种族值以外，由玩家额外配置的属性成长值。

规则暂定为：

- 培养值范围：0 到 10
- 每只宠物最多可以有三维拥有培养值
- 培养值会影响最终实际属性
- 不同玩家的培养值配置可能不同

需要数据库或公式支持：

```text
实际属性 = 种族值 + 性格修正 + 培养值修正
```

具体取整规则、培养值换算公式需要后续确认。

## 3. 性格

性格会影响宠物六维属性。

当前规则：

- 性格会使六维中的其中一维获得 20% 增益
- 性格会使六维中的其中一维获得 20% 减益
- 其他属性不变

例如：

```text
某性格：
物攻 × 1.2
魔攻 × 0.8
其他属性 × 1.0
```

需要确认：

- 取整发生在性格修正前还是修正后

## 4. 实际属性

实际属性是战斗中真正进入伤害公式的属性。

包括：

- 生命
- 物攻
- 物防
- 魔攻
- 魔防
- 速度

敌方实际属性由以下因素决定：

```text
宠物种族值
+ 等级
+ 性格
+ 培养值
+ 战斗中的 Buff/Debuff
+ 天气、异常状态、技能效果等临时修正
```

系统的目标是推算敌方的基础配置，并在战斗中结合实时状态计算当前有效属性。

# 四、对战流程

## 1. 准备阶段

对战开始时进入准备页面。

此时系统可以看到双方出场的六只宠物。

系统需要完成以下任务：

### 己方信息读取

己方六只宠物由玩家提前录入完整信息。

包括：

- 宠物名称
- 种族值
- 性格
- 培养值
- 技能组
- 等级，固定为 60 级
- 实际属性
- 技能信息

### 敌方信息建档

敌方六只宠物的种类在准备页面可见。

系统通过识别或手动输入确认敌方六只宠物名称，并从数据库中读取：

- 敌方宠物种族值
- 宠物属性类型
- 可能技能池
- 常见性格配置
- 常见培养值配置
- 默认候选配置集合

此时敌方每只宠物处于“配置未知”状态。

系统需要为敌方每只宠物建立一个候选配置集合：

```text
敌方宠物A：
- 候选配置1：性格A + 培养值方案A
- 候选配置2：性格A + 培养值方案B
- 候选配置3：性格B + 培养值方案A
- ...
```

后续战斗中的每次伤害都会用于过滤这个候选集合。

------

## 2. 战斗阶段

进入实际对战后，系统需要持续识别和记录：

- 当前我方上场宠物
- 当前敌方上场宠物
- 双方当前生命值
- 双方 Buff/Debuff
- 天气
- 异常状态
- 技能使用情况
- 伤害数值
- 能量变化
- 宠物切换情况

每次发生伤害时，系统生成一条战斗事件记录。

------

## 3. 信息收敛阶段

系统不应假设一次伤害就能准确反推出敌方配置。

正确逻辑应为：

```text
初始化敌方可能配置集合
↓
发生一次伤害
↓
根据伤害公式过滤不可能配置
↓
保留匹配配置
↓
发生下一次伤害
↓
继续过滤
↓
候选配置越来越少
↓
最终得到高置信度配置
```

当候选配置数量很少，或者多个候选配置的实际伤害计算结果差异很小，系统可以标记为：

- 未知
- 低置信度
- 中置信度
- 高置信度
- 已基本确认

------

# 五、伤害计算需求

## 1. 伤害计算的展示方式

所有伤害计算都需要同时展示两种结果：

```text
准确伤害值
生命百分比
```

例如：

```text
技能A：
准确伤害：128
生命百分比：42.7%
```

原因：

- 实际战斗中伤害会显示准确数值
- 但敌方当前生命通常只能通过百分比血量观察
- 对我方来说，己方最大生命值和当前生命值已知，因此可以精确计算敌方技能对我方造成的整数伤害和百分比伤害
- 对敌方来说，敌方最大生命值可能随着配置不同而变化，因此百分比伤害也可能是一个区间

因此，对敌方伤害展示可能是：

```text
技能A：
准确伤害：118 ~ 126
生命百分比：38.2% ~ 41.6%
```

当敌方配置被确认后，展示可以变为：

```text
技能A：
准确伤害：122
生命百分比：40.1%
```

------

## 2. 我方攻击敌方的伤害计算

输入：

- 我方当前宠物
- 我方实际属性
- 我方技能
- 我方 Buff/Debuff
- 敌方当前宠物
- 敌方候选配置
- 敌方 Buff/Debuff
- 天气
- 异常状态
- 增伤/减伤状态
- 属性克制关系
- 技能特殊效果

输出：

- 每个技能对敌方当前宠物的准确伤害值
- 每个技能对敌方当前宠物的生命百分比
- 如果敌方配置未知，则输出区间
- 如果敌方配置已确认，则输出单值
- 对敌方后备宠物也可以计算预估伤害

示例：

```text
我方当前宠物：A
敌方当前宠物：B

技能1：
准确伤害：96 ~ 108
生命百分比：31.4% ~ 35.9%

技能2：
准确伤害：142 ~ 158
生命百分比：46.2% ~ 52.4%

技能3：
准确伤害：无法有效造成伤害，可能被抵抗
生命百分比：8.1% ~ 10.3%
```

------

## 3. 敌方攻击我方的伤害计算

输入：

- 敌方当前宠物
- 敌方候选配置
- 敌方已知技能或可能技能池
- 敌方 Buff/Debuff
- 我方当前宠物
- 我方实际属性
- 我方 Buff/Debuff
- 天气
- 异常状态
- 属性克制关系
- 技能特殊效果

输出：

- 敌方已知技能对我方当前宠物的准确伤害值
- 敌方已知技能对我方当前宠物的生命百分比
- 敌方可能技能对我方当前宠物的伤害区间
- 敌方技能对我方后备宠物的伤害区间

示例：

```text
敌方当前宠物：B
我方当前宠物：A

已确认技能：
技能X：
准确伤害：132 ~ 146
生命百分比：44.0% ~ 48.7%

可能技能：
技能Y：
准确伤害：88 ~ 102
生命百分比：29.3% ~ 34.0%

技能Z：
准确伤害：170 ~ 196
生命百分比：56.7% ~ 65.3%
```

------

# 六、敌方配置推算需求

## 1. 敌方配置推算目标

系统需要推算的不是敌方种族值，而是敌方具体配置。

包括：

- 性格
- 培养值分配
- 实际生命
- 实际物攻
- 实际物防
- 实际魔攻
- 实际魔防
- 实际速度
- 技能组
- 当前 Buff/Debuff
- 当前状态修正

------

## 2. 候选配置集合

每只敌方宠物在准备阶段生成候选配置集合。

候选配置由以下内容组成：

```text
BuildCandidate {
  性格
  培养值分配
  实际生命
  实际物攻
  实际物防
  实际魔攻
  实际魔防
  实际速度
  匹配分数
  是否已排除
  排除原因
}
```

例如：

```text
敌方宠物B 候选配置：

候选1：
性格：物攻+20%，魔攻-20%
培养值：生命10 / 物攻10 / 速度10
匹配状态：保留

候选2：
性格：魔攻+20%，物攻-20%
培养值：生命10 / 魔攻10 / 速度10
匹配状态：已排除
排除原因：敌方技能X造成伤害过高，不符合该魔攻配置

候选3：
性格：物防+20%，速度-20%
培养值：生命10 / 物防10 / 魔防10
匹配状态：保留
```

------

## 3. 我方造成伤害时的反推逻辑

当我方攻击敌方造成伤害时，系统可以反推敌方防御侧配置。

反推内容包括：

- 敌方生命值
- 敌方物防
- 敌方魔防
- 敌方防御相关性格
- 敌方防御相关培养值
- 敌方是否存在减伤状态
- 敌方是否受到天气、异常状态影响

流程：

```text
记录我方技能
↓
读取我方攻击属性
↓
读取技能威力、技能类型、属性克制
↓
读取双方 Buff/Debuff、天气、异常状态
↓
枚举敌方所有候选配置
↓
计算每个候选配置下的理论伤害
↓
与实际伤害对比
↓
保留匹配候选
↓
排除不匹配候选
```

------

## 4. 敌方造成伤害时的反推逻辑

当敌方攻击我方造成伤害时，系统可以反推敌方攻击侧配置。

反推内容包括：

- 敌方物攻
- 敌方魔攻
- 敌方攻击相关性格
- 敌方攻击相关培养值
- 敌方可能使用的技能
- 敌方是否存在增伤状态
- 敌方是否受到天气、异常状态影响

如果敌方技能已识别：

```text
敌方技能已知
↓
枚举敌方候选配置
↓
计算每个配置下的理论伤害
↓
与实际伤害对比
↓
过滤候选配置
```

如果敌方技能未知：

```text
敌方技能未知
↓
读取该宠物可能技能池
↓
枚举可能技能
↓
枚举敌方候选配置
↓
计算每个“技能 + 配置”的理论伤害
↓
与实际伤害对比
↓
保留可能组合
```

展示示例：

```text
敌方本次攻击可能情况：

高匹配：
技能A + 物攻性格 + 物攻培养10
理论伤害：132
实际伤害：134
误差：2

中匹配：
技能B + 魔攻性格 + 魔攻培养10
理论伤害：128
实际伤害：134
误差：6

已排除：
技能C
排除原因：理论最高伤害仅为91，无法解释实际伤害134
```

# 七、战斗事件日志

系统必须记录每一次关键战斗事件。

尤其是每一次伤害事件，都需要保存当时的完整快照。

## 1. 伤害事件记录

```text
DamageEvent {
  回合编号
  时间戳
  攻击方宠物
  防御方宠物
  使用技能
  技能是否确认
  伤害准确值
  防御方血量变化百分比
  攻击方 Buff/Debuff 快照
  防御方 Buff/Debuff 快照
  天气快照
  异常状态快照
  增伤状态快照
  减伤状态快照
  属性克制结果
  是否存在护盾
  是否存在特殊技能效果
  信息来源：自动识别 / 手动输入
  识别置信度
}
```

## 2. 为什么需要事件快照

因为 Buff、天气、异常状态、增伤、减伤等信息会随着战斗变化。

一次伤害发生时的状态，可能和当前画面中的状态不同。

如果不记录伤害发生时的快照，后续推算会把旧伤害错误套用到新状态上，导致配置反推错误。

------

# 八、图像识别与手动输入需求

## 1. 自动识别内容

系统需要通过图像识别或 OCR 捕获：

- 准备阶段双方六只宠物
- 当前对位宠物
- 宠物头像
- 技能名称
- 伤害数值
- 生命百分比或血条变化
- Buff/Debuff 图标
- 天气状态
- 异常状态
- 增伤/减伤状态
- 能量变化
- 宠物切换信息

## 2. 手动输入优先级

图像识别可能失败，因此必须允许手动补充。

规则：

```text
手动输入优先级 > 自动识别结果
```

手动输入适用于：

- 识别错误的宠物名称
- 识别错误的技能名称
- 识别错误的伤害数值
- Buff/Debuff 叠层识别失败
- 天气状态识别失败
- 异常状态识别失败
- 敌方技能使用后手动确认
- 准备阶段手动确认双方阵容

## 3. 输入效率优化

手动输入需要尽可能快。

建议支持：

- 宠物名称关键字搜索
- 技能名称关键字搜索
- 常见技能一键选择
- Buff/Debuff 图标快速选择
- 天气状态快速选择
- 异常状态快速选择
- 最近使用项
- 战斗事件快速纠错

------

# 九、数据需求

## 1. 宠物数据库

每只宠物需要记录：

- 宠物名称
- 宠物头像
- 属性类型
- 六维种族值
- 可学习技能列表
- 常见技能组
- 常见性格
- 常见培养值方案
- 进化或形态信息
- 图像识别模板

## 2. 技能数据库

每个技能需要记录：

- 技能名称
- 技能图标
- 技能属性
- 技能类别：物理 / 魔法 
- 基础威力
- 能量消耗
- 不考虑命中率
- 是否固定伤害
- 是否百分比伤害
- 是否无视防御
- 是否多段伤害
- 是否吸血
- 是否附加异常
- 是否附加 Buff/Debuff
- 是否受天气影响
- 是否受状态影响
- 是否有特殊增伤/减伤规则
- 图像识别模板

## 3. 状态数据库

需要记录所有可能影响伤害的状态：

- Buff
- Debuff
- 天气
- 异常状态
- 免疫
- 克制
- 抵抗
- 其他特殊战场效果

每个状态需要记录：

```text
状态名称
状态类型
影响对象
影响属性
倍率
持续时间
是否可叠加
最大层数
是否影响伤害公式
是否影响属性计算
```

# 十、信息展示需求

系统需要在独立程序页面展示所有核心信息。

## 1. 当前对位信息

展示：

```text
我方当前宠物
敌方当前宠物
天气
双方异常状态
双方 Buff/Debuff
双方能量
双方生命状态
敌方配置置信度
```

## 2. 我方技能对敌方伤害

每个技能展示：

```text
技能名称
准确伤害值
生命百分比
是否克制
是否受天气影响
是否存在区间
伤害置信度
```

示例：

```text
我方技能伤害预测：

准确伤害：120 ~ 136

```

展示：

```text

准确伤害值
生命百分比
是否克制
是否可能击杀
对我方后备宠物的伤害
```

示例：

```text
敌方技能伤害预测：

已确认技能X：

准确伤害：142 ~ 156
生命百分比：47.3% ~ 52.0%

对后备宠物1：
```

# 十三、边界条件与当前假设

当前版本先基于以下假设设计：

1. 等级固定为 60 级。
2. 敌方六只宠物在准备阶段可见。
3. 己方六只宠物完整配置由玩家提前输入。
4. 敌方宠物种族值可由宠物名称从数据库读取。
5. 敌方未知配置主要包括性格、培养值、技能组。
6. 培养值范围为 0 到 10。
7. 每只宠物最多三维有培养值。
8. 性格会使一维 +20%，一维 -20%。
9. 伤害结果需要同时展示准确伤害值和生命百分比。
10. 减伤、增伤、天气、异常状态需要进入伤害计算。
11. Buff/Debuff 需要进入伤害计算。
12. 图像识别结果可以被手动输入覆盖。
13. 敌方配置推算采用候选集合过滤方式，而不是一次性直接反推。
14. 如果敌方技能未知，系统需要枚举该宠物可能技能池进行联合推算。
15. 所有计算结果在独立页面展示。

# 十四、仍需进一步确认的关键规则

为了进入开发设计，后面还需要补充这些规则：

## 1. 属性计算公式

需要明确：

```text
种族值、培养值、性格如何计算成最终属性
```

尤其是：

- 培养值 1 点对应多少实际属性
- 生命和其他五维是否使用同一套公式
- 性格 20% 修正的取整规则
- 培养值和性格的计算先后顺序

## 2. 伤害公式

需要明确完整伤害公式。

包括：

- 等级项
- 技能威力
- 攻击属性
- 防御属性
- 属性克制倍率
- Buff/Debuff 倍率
- 天气倍率
- 异常状态倍率
- 最终取整规则

## 3. 取整规则

这是反推准确度的关键。

需要确认：

- 每一步是否取整
- 是向下取整、四舍五入还是向上取整
- 百分比血量是否四舍五入
- 战斗显示伤害是否为最终整数

## 4. 技能特殊规则

需要整理特殊技能，例如：

- 固定伤害
- 百分比伤害
- 无视防御
- 多段伤害
- 追加伤害
- 根据自身生命变化的伤害
- 根据敌方生命变化的伤害
- 根据天气变化的伤害
- 根据异常状态变化的伤害
- 护盾
- 免疫
- 吸血
- 反伤

这些技能不能简单套普通伤害公式。

## 5. 天气、异常、增伤、减伤规则

需要明确：

- 天气有哪些
- 每种天气影响哪些属性或技能
- 异常状态有哪些
- 异常状态是否影响攻击、防御、速度、伤害
- 多个倍率的乘法顺序
- 是否存在上限或下限

# 十五、开发优先级建议

## 第一阶段：纯手动输入版本

先不做图像识别，先做核心计算逻辑。

目标：

- 手动录入双方阵容
- 手动录入己方完整配置
- 敌方根据数据库生成候选配置
- 手动输入技能、伤害、状态
- 系统完成候选配置过滤
- 展示双方技能伤害

这是最重要的 MVP。

## 第二阶段：半自动识别版本

加入部分图像识别。

目标：

- 自动识别当前双方宠物
- 自动识别伤害数字
- 自动识别技能名称
- 自动识别 Buff/Debuff 图标
- 允许手动修正

## 第三阶段：实时辅助版本

目标：

- 战斗中自动刷新
- 自动维护敌方配置候选集合
- 自动展示当前最优伤害数据
- 支持场下宠物伤害预估
- 支持切换宠物前的伤害预览

## 第四阶段：高级推算版本

目标：

- 根据多次伤害自动判断敌方技能组
- 根据伤害误差自动提示可能遗漏的减伤/增伤状态
- 根据敌方行动习惯更新技能概率
- 支持更多特殊技能和特殊状态

# 十六、总结版核心需求描述

本系统的核心需求是：

> 在洛克王国世界 PVP 对战中，系统通过准备阶段获取双方六只宠物信息。己方宠物完整配置由玩家提前输入，敌方宠物仅确认种类，具体性格、培养值和技能组未知。系统根据数据库为敌方每只宠物生成可能配置集合，并在战斗过程中通过双方造成的实际伤害、血量百分比变化、技能信息、Buff/Debuff、天气、异常状态、增伤/减伤等信息，不断过滤和收敛敌方候选配置。最终系统实时展示我方技能对敌方的准确伤害值和生命百分比，以及敌方已知或可能技能对我方的准确伤害值和生命百分比，为玩家提供可靠的战斗数据支持。
