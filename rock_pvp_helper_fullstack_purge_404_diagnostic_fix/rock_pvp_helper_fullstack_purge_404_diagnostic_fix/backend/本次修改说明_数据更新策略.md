# 本次修改说明：数据更新策略与管理接口

## 修改目标

根据当前项目状态，后端不再依赖 `seed` 作为正式数据初始化方式；精灵、技能等规则数据以你已经爬取并导入的全量数据库为准。后端启动时也不主动检查远程爬虫更新，避免外部网络影响本地战斗功能。

本次代码补齐了主动式数据更新管理能力：

```text
后端启动
↓
不检查远程，不自动爬虫，不自动写库
↓
用户需要更新时，主动调用 admin 接口
↓
先 check 或 dry-run
↓
确认后 commit=true 写库
```

## 本次新增 / 修改内容

### 1. 新增远程检查接口

```http
POST /api/v1/admin/data-updates/rocom/check
```

用途：只抓取 BWIKI 图鉴列表页，与本地数据库中 `rocom_elf_*` 精灵 ID 做差异对比。

特点：

- 不抓详情页。
- 不清洗数据。
- 不写数据库。
- 适合快速判断“是否可能有新增精灵”。

示例请求：

```json
{
  "limit": 0,
  "include_new_elves_limit": 100
}
```

示例响应核心字段：

```json
{
  "status": "changed",
  "remote_count": 523,
  "local_rocom_count": 520,
  "new_elf_count": 3,
  "new_elves": []
}
```

说明：该接口只能判断列表层面的新增或缺失；详情页字段变化仍需要执行 dry-run 同步或本地导入来确认。

### 2. 新增本地 cleaned JSON 导入接口

```http
POST /api/v1/admin/data-updates/rocom/import-local
```

用途：不访问远程，只把已经生成的 cleaned JSON 导入数据库。

适用场景：

```text
你已经通过爬虫得到 data/rocom/cleaned/*.json
↓
只需要重新导入数据库
↓
不想重新访问 BWIKI
```

示例 dry-run：

```json
{
  "commit": false,
  "cleaned_dir": "../data/rocom/cleaned"
}
```

确认后提交：

```json
{
  "commit": true,
  "cleaned_dir": "../data/rocom/cleaned",
  "data_version": "rocom_bwiki_20260516"
}
```

### 3. 保留远程同步接口

```http
POST /api/v1/admin/data-updates/rocom/sync
```

该接口继续执行：

```text
远程爬取 → 清洗 → dry-run 或提交入库
```

默认 `commit=false`，不会写库。确认后再传 `commit=true`。

### 4. 任务查询接口支持不同任务类型

```http
GET /api/v1/admin/data-updates/rocom/jobs/{job_id}
GET /api/v1/admin/data-updates/rocom/jobs
```

现在任务结果中会包含：

```text
job_type = sync / import_local
```

### 5. README 已更新

README 中已经改为：

- 正式流程不再要求执行 `python -m app.seed.minimal_seed`。
- `minimal_seed` 只作为空库开发测试用途。
- 数据更新推荐走主动 admin 接口。
- 后端启动默认不检查、不爬虫、不写库。

```env

ROCOM_AUTO_UPDATE_COMMIT=false

```


不需要执行 seed。

## 当前推荐数据更新流程

### 第一步：检查远程是否可能有新增

```bash
curl -X POST http://localhost:8000/api/v1/admin/data-updates/rocom/check \
  -H "Content-Type: application/json" \
  -d '{"limit": 0, "include_new_elves_limit": 100}'
```

### 第二步：执行 dry-run

如果你要重新爬远程：

```bash
curl -X POST http://localhost:8000/api/v1/admin/data-updates/rocom/sync \
  -H "Content-Type: application/json" \
  -d '{"commit": false, "limit": 0, "force": false}'
```

如果你已有 cleaned JSON：

```bash
curl -X POST http://localhost:8000/api/v1/admin/data-updates/rocom/import-local \
  -H "Content-Type: application/json" \
  -d '{"commit": false, "cleaned_dir": "../data/rocom/cleaned"}'
```

### 第三步：查询任务结果

```bash
curl http://localhost:8000/api/v1/admin/data-updates/rocom/jobs/{job_id}
```

### 第四步：确认后提交

```bash
curl -X POST http://localhost:8000/api/v1/admin/data-updates/rocom/import-local \
  -H "Content-Type: application/json" \
  -d '{"commit": true, "cleaned_dir": "../data/rocom/cleaned"}'
```
