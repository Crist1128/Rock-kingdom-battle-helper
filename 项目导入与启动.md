# 项目导入与启动

## 一、项目结构

假设项目结构是：

```text
Rock-kingdom-battle-helper/
  backend/
  frontend/
```

需要启动两个服务：

```text
后端：FastAPI + SQLite
前端：React + Vite
```

数据库为空时，需要先通过前端“数据管理”主动导入或同步规则数据。

------

## 二、准备后端 conda 环境

进入项目后端目录：

```bash
cd backend
```

创建 conda 环境：

```bash
conda create -n rock-pvp-helper python=3.12 -y
```

激活环境：

```bash
conda activate rock-pvp-helper
```

升级 pip：

```bash
python -m pip install --upgrade pip
```

安装后端依赖：

```bash
python -m pip install -e ".[dev]"
```

复制环境变量文件。

Windows CMD：

```bash
copy .env.example .env
```

PowerShell：

```powershell
Copy-Item .env.example .env
```

macOS / Linux：

```bash
cp .env.example .env
```

------

## 三、初始化空数据库结构

数据库为空时，先执行迁移：

```bash
python -m alembic upgrade head
```

这一步只创建表结构，不会自动写入精灵数据。

启动后端：

```bash
python -m uvicorn app.main:app --reload
```

后端地址：

```text
http://127.0.0.1:8000
```

接口文档：

```text
http://127.0.0.1:8000/docs
```

保持这个终端不要关闭。

------

## 四、启动前端

新开一个终端，进入前端目录：

```bash
cd frontend
```

安装依赖：

```bash
npm install --no-audit --no-fund
```

启动前端：

```bash
npm run dev
```

前端地址：

```text
http://127.0.0.1:5173
```

打开浏览器访问这个地址。

------

## 五、数据库为空时，如何导入规则数据

进入前端页面后：

```text
设置 → 数据管理 → 静态规则数据库更新
```

如果后端 `.env` 里配置了：

```env
ADMIN_UPDATE_TOKEN=你的token
```

那么前端页面里也要填写相同的 Admin Token。

如果没有配置，可以留空。

### 推荐操作顺序

先点击：

```text
检查远程更新
```

然后优先尝试：

```text
本地导入 dry-run
```

如果本地已经有爬虫清洗后的 cleaned JSON 数据，dry-run 会显示预计新增、更新、跳过、警告等结果。

确认没有问题后，再点击：

```text
本地导入并写库
```

如果本地没有 cleaned JSON，可以使用：

```text
远程同步 dry-run
```

这会执行：

```text
远程爬取 → 清洗 → 预览导入结果
```

确认结果正常后，再点击：

```text
远程同步并写库
```

------

## 六、日常启动项目

完成一次数据库导入后，以后日常启动只需要：

### 终端 1：后端

```bash
conda activate rock-pvp-helper
cd backend
python -m uvicorn app.main:app --reload
```

### 终端 2：前端

```bash
cd frontend
npm run dev
```

不需要每次启动都跑爬虫，也不需要每次执行 seed。

------

## 七、重要说明

`minimal_seed` 只是空库开发测试用，不是正式初始化方式。

正式数据应该通过：

```text
设置 → 数据管理 → 静态规则数据库更新
```

主动导入。

爬虫不是常驻服务，不需要单独启动。它只在你点击前端的数据更新按钮，或者调用后端 admin 数据更新接口时才运行。

后端默认不应该自动爬虫、自动检查远程、自动写库。建议 `.env` 保持：

```env
ROCOM_AUTO_UPDATE_ON_STARTUP=false
ROCOM_AUTO_UPDATE_COMMIT=false
ROCOM_AUTO_UPDATE_FORCE=false
```

## 八、常见问题

### 1. 前端显示没有精灵数据

说明数据库还没有导入规则数据。去：

```text
设置 → 数据管理 → 静态规则数据库更新
```

执行本地导入或远程同步。

### 2. 后端启动成功，但前端请求失败

确认后端地址是：

```text
http://127.0.0.1:8000
```

确认前端地址是：

```text
http://127.0.0.1:5173
```

Vite 会把 `/api/v1/...` 代理到后端。

### 3. npm install 很慢或卡住

可以设置 npm 镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm config set audit false
npm config set fund false
npm install --no-audit --no-fund
```

### 4. conda 环境里找不到 alembic

确认你已经激活环境并安装依赖：

```bash
conda activate rock-pvp-helper
cd backend
python -m pip install -e ".[dev]"
python -m alembic upgrade head
```

### 5. 数据更新写库失败

先用 dry-run 看错误，不要直接提交写库。优先检查：

```text
Admin Token 是否正确
数据库是否已经迁移到最新
后端终端是否有爬虫或导入报错
本地 cleaned JSON 路径是否存在
```

## 最短启动版

```bash
# 后端
cd backend
conda create -n rock-pvp-helper python=3.12 -y
conda activate rock-pvp-helper
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
copy .env.example .env
python -m alembic upgrade head
python -m uvicorn app.main:app --reload
```

另开终端：

```bash
# 前端
cd frontend
npm install --no-audit --no-fund
npm run dev
```

然后打开：

```text
http://127.0.0.1:5173
```

数据库为空时，在页面里进入：

```text
设置 → 数据管理 → 静态规则数据库更新
```

先 dry-run，再提交写库。
