# Frontend MVP 范围说明

## 已实现页面

1. 战斗首页
   - 创建战斗
   - 手动添加 battle_id
   - localStorage 最近战斗记录
   - 后端 health 检测

2. 己方配置管理
   - 查询己方配置
   - 创建 / 编辑己方配置
   - 精灵搜索、性格选择、技能搜索、个体资质录入
   - 面板属性只展示后端返回缓存，不做前端真实计算

3. 准备阶段
   - 录入我方 6 个配置槽
   - 录入敌方 6 个精灵槽
   - 设置双方首发
   - 调用 lineup 接口生成敌方候选
   - 调用 start 接口进入 battle 阶段

4. 战斗工作台
   - 双方队伍区
   - 当前对位面板
   - 快捷录入：伤害、资源、状态、切换
   - 统一状态分组展示
   - 候选摘要 / 速度分布展示
   - 时间线简版展示

5. 事件日志
   - battle_id 切换
   - 时间线完整展示
   - 纠错/作废/重放的待后端提示

6. 规则库
   - 精灵、技能、状态、性格只读查询

7. 设置
   - API 地址显示
   - health 检测
   - 导入/导出/备份待后端接口提示

## 真实计算处理原则

本项目当前真实计算部分尚未实现，因此前端遵循以下约束：

- 不在前端计算真实伤害。
- 不在前端计算真实速度先手概率。
- 不展示伪伤害区间。
- 不根据占位公式排除候选。
- 候选面板保留 `formula_unavailable` 提示。
- 技能伤害、击杀判断、速度概率只显示“未实现”。
