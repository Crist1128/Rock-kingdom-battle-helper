# 本次修改说明：前端主动数据更新入口

## 修改目标

当前后端已经提供 `admin/data-updates/rocom/*` 数据更新接口，但前端没有入口。对于刚 clone 项目、数据库为空或需要更新新精灵数据的用户，需要能在前端主动触发数据更新。

本次只修改前端，不修改后端。

## 已新增功能

在 `设置 / 数据管理` 页面新增 **静态规则数据库更新** 区域，包含：

1. **检查远程更新**
   - 调用 `POST /api/v1/admin/data-updates/rocom/check`
   - 只解析 BWIKI 图鉴列表，不爬详情、不写库。
   - 显示远程数量、本地 rocom 数量、新增精灵数量、新增预览。

2. **从本地 cleaned JSON 导入**
   - 调用 `POST /api/v1/admin/data-updates/rocom/import-local`
   - 支持 dry-run 和提交写库。
   - 支持填写 `cleaned_dir` 和可选 `data_version`。

3. **远程爬取、清洗并导入**
   - 调用 `POST /api/v1/admin/data-updates/rocom/sync`
   - 支持 dry-run 和提交写库。
   - 支持 limit、delay、data_version、force、with_images、write_artifacts。

4. **数据更新任务列表**
   - 调用 `GET /api/v1/admin/data-updates/rocom/jobs`
   - 运行中任务每 3 秒自动刷新。
   - 可查看任务参数和结果 JSON。

5. **管理令牌复用**
   - 页面顶部统一填写 `Admin Token`。
   - 同时用于数据更新接口和归档战斗物理清理接口。
   - 前端只保存到浏览器 localStorage，不把 token 写入源码。

## 使用建议

新 clone 项目、数据库为空时，推荐顺序：

```text
1. 启动后端和前端。
2. 打开 设置 / 数据管理。
3. 填写 Admin Token（如果后端配置了 ADMIN_UPDATE_TOKEN）。
4. 点击“检查更新”。
5. 优先尝试“本地导入 dry-run”；如果没有本地 cleaned JSON，再使用“远程同步 dry-run”。
6. 确认 dry-run 结果正常后，再点击“提交本地导入”或“提交远程同步”。
7. 任务成功后，前端会刷新精灵、技能、状态、性格和己方配置相关缓存。
```

## 注意

- 爬虫不是常驻服务，也不会随后端启动自动运行。
- `sync` 会访问外部 BWIKI，受网络和页面结构影响；推荐先 dry-run。
- `import-local` 不访问远程，更适合已经生成 cleaned JSON 的情况。
- 真实战斗公式、候选过滤、速度计算仍未实现，本次不涉及。
