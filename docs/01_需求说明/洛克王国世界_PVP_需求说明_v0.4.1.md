## 一、项目概述

本项目开发一个用于 **洛克王国世界 PVP 对战** 的战斗信息获取、敌方配置推算与伤害计算工具。

系统不自动替玩家做战斗决策，而是在对战过程中收集事实、维护状态、计算伤害、推算敌方配置，并把关键结果实时展示给玩家。

核心目标：

```text
准备阶段识别双方六只精灵
↓
读取己方完整配置
↓
为敌方精灵生成候选配置集合
↓
战斗中记录技能、伤害、生命变化、能量变化、状态图标、天气、印记、异常、连击等事实
↓
基于面板属性、伤害公式、状态快照计算理论伤害
↓
用实际伤害和扣血百分比过滤敌方候选配置
↓
实时展示伤害区间、先手关系、击杀判断和敌方配置置信度
```

系统的核心不是“推荐出招”，而是：

- 收集战斗事实。
- 维护当前战斗状态。
- 根据伤害公式计算结果。
- 根据实际伤害收敛敌方候选配置。
- 以可解释方式展示计算依据和不确定性。

### 2.1 精灵英文缩写

从 v0.4.1 起，所有代码字段、结构名、数据库表名中的精灵英文词条统一使用：

```text
elf
```

正式命名示例：

| 用途 | 正式命名 |
|---|---|
| 精灵唯一 ID | `elf_id` |
| 精灵名称 | `elf_name` |
| 精灵静态定义 | `ElfDefinition` |
| 敌方精灵运行时状态 | `EnemyElfState` |
| 战斗内精灵状态表 | `battle_elf_state` |
| 状态归属精灵 ID | `owner_elf_id` |
| 事件行动方精灵 ID | `actor_elf_id` |
| 事件目标精灵 ID | `target_elf_id` |
| 精灵切换事件 | `switch_elf` |

### 2.2 精灵基础身份

精灵静态身份由以下字段确定：

```text
elf_id
elf_name
avatar
```

规则：

- 精灵有唯一 ID。
- 精灵有标准名称。
- 精灵没有别名字段。
- 精灵有头像。
- 代码内部引用精灵必须使用 `elf_id`。
- UI 展示使用 `elf_name` 和 `avatar`。
- 图像识别优先使用头像、名称区域和手动确认，不依赖精灵别名表。
- 技能可以保留 `alias_names`，用于 OCR 误识别和搜索容错；这个规则不适用于精灵。

## 三、核心业务流程

### 3.1 准备阶段

对战开始前进入准备页面，系统可以看到双方各六只精灵。

己方六只精灵由玩家提前录入完整配置，因此己方信息是确定的。

己方需要维护：

- `elf_id`。
- 精灵名称和头像。
- 系别类型。
- 六维种族资质。
- 个体资质。
- 性格。
- 技能组。
- 面板属性。
- 当前生命值 / 百分比。
- 当前能量。
- 当前状态效果列表。

敌方在准备阶段只能确认精灵种类。

敌方准备阶段可确定：

- `elf_id`。
- 精灵名称和头像。
- 系别类型。
- 六维种族资质。
- 可学习技能池。
- 常见技能组、常见性格、常见个体资质分布。

敌方准备阶段未知：

- 性格。
- 个体资质维度。
- 个体资质具体数值。
- 最终面板属性。
- 实际携带技能组。
- 战斗中产生的状态变化。

系统需要为敌方每只精灵建立候选配置集合。常见配置只影响候选权重，不用于直接排除冷门配置。

### 3.2 战斗阶段

进入实际对战后，系统持续记录：

- 当前我方上场精灵。
- 当前敌方上场精灵。
- 双方生命状态。
- 双方能量。
- 双方技能使用情况。
- 双方当前所有状态图标。
- 天气 / 战场状态。
- 双方队伍侧印记列表。
- 防御技能状态。
- 增伤、减伤、威力变化、能耗变化、连击数变化等效果。
- 状态驱散、转换、转移、继承、切换清除事件。
- 精灵切换事件。
- 单次伤害数值。
- 动画多段伤害最终显示的总伤害。
- 连击的单段伤害和连击次数。
- 敌方扣血百分比。
- 血量百分比变化。

每次发生可靠伤害事件后，系统用该事件过滤敌方候选配置。

### 3.3 信息收敛阶段

敌方配置通过多次战斗证据逐步收敛。

```text
准备阶段生成候选配置集合
↓
发生技能事件、伤害事件、状态变化事件
↓
读取事件发生瞬间的 BattleEffectSnapshot
↓
枚举敌方候选配置
↓
形成 DamageFormulaContext
↓
计算理论伤害
↓
与实际伤害和扣血百分比对比
↓
排除不可能配置
↓
保留匹配配置并更新权重
↓
输出配置置信度、速度区间、伤害区间和解释
```

敌方配置状态分为：

| 状态 | 含义 |
|---|---|
| 未知 | 候选集合很大，缺少有效伤害证据 |
| 低置信度 | 已经排除部分候选，但仍存在多种可能 |
| 中置信度 | 候选范围明显缩小，可给出较稳定区间 |
| 高置信度 | 只剩少量候选，伤害与速度预测较可靠 |
| 已基本确认 | 候选几乎唯一，后续主要处理状态变化 |

## 四、核心概念与规则

### 4.1 培养配置

培养配置定义为：

```text
培养配置 = 性格 + 个体资质分布
```

敌方未知配置主要包括：

- 性格。
- 个体资质存在的维度。
- 各个体资质维度的数值。
- 技能组。

### 4.2 个体资质

当前规则：

- 单只精灵一定有 1 到 3 个维度存在个体资质。
- 通常情况下，单只精灵存在个体资质的维度数量为 3 个。
- 存在个体资质的维度，数值固定范围为 7 到 10。
- 没有个体资质的维度，按 0 计算。
- 个体资质参与面板属性计算。
- 所有 PVP 精灵统一按照 6 星、0 觉醒、50 成长等级计算。

### 4.3 性格

性格规则：

- 必定存在一个正面修正属性。
- 必定存在一个负面修正属性。
- 正面修正与负面修正不会是同一维。
- 正面修正：该维度属性 +20%。
- 负面修正：该维度属性 -10%。
- 其他属性不变。
- 生命、物攻、物防、魔攻、魔防、速度均可能被性格修正。
- 性格不影响成长值。
- 性格修正后再进行取整。

### 4.4 面板属性

面板属性是精灵进入战斗前的最终六维属性。

包括：

- 生命。
- 物攻。
- 物防。
- 魔攻。
- 魔防。
- 速度。

面板属性由种族资质、个体资质、等级、性格、成长值和取整规则计算得到。

敌方候选配置保存的是面板属性，不保存带状态修正后的临时属性。

### 4.5 伤害公式上下文

不再把“战斗有效属性”作为核心概念。

伤害计算使用：

```text
DamageFormulaContext = 面板属性 + 技能规则 + 属性克制 + BattleEffectSnapshot + 伤害公式 + 特殊公式处理器
```

状态、天气、印记、异常、技能效果等不提前合并成固定属性，而是在公式中作为修正项、条件项或特殊处理器参与计算。

调试或解释时可以临时输出某次计算中的修正过程，但这些临时值不进入敌方候选配置。

### 4.6 速度与先手上下文

速度判断使用：

```text
SpeedContext = 面板速度 + 当前影响速度或行动顺序的状态效果 + 技能先手规则
```

速度判断分两层：

1. 行动规则修正：先手、迅捷、蓄力、无法行动、冰冻、眩晕、打断、无法更换等。
2. 常规速度比较。

常规速度规则：

```text
if 我方速度 > 敌方速度:
    我方常规先手
elif 我方速度 < 敌方速度:
    敌方常规先手
else:
    双方同速，各 50% 概率先手
```

敌方速度未确认时，根据候选配置计算综合先手概率。

## 五、统一状态效果系统

### 5.1 设计原则

所有会在游戏状态栏、天气栏、印记栏、技能图标或战斗 UI 中显示，并且会影响后续判断或计算的效果，统一进入：

```text
BattleEffectSystem
```

不要拆成多套互不相干的状态系统。

```text
不推荐：BuffSystem / DebuffSystem / AbnormalSystem / WeatherSystem / MarkSystem
推荐：BattleEffectSystem + 字段区分规则
```

统一状态系统用于：

- 一次性识别所有状态图标。
- 用同一个状态选择器进行手动修正。
- 用统一事件记录状态获得、移除、驱散、转换、转移、继承、切换清除。
- 在伤害、速度、连击、能耗、威力等计算中读取同一份状态快照。

### 5.2 状态规则由字段决定

`category` 是展示和检索标签，不是规则本身。

状态是否清除、是否叠层、是否参与公式、是否能被驱散或转换，都应由字段决定。

关键字段包括：

- `category`：分类标签。
- `owner_scope`：归属范围。
- `clear_on_switch`：切换是否清除。
- `stack_rule`：叠层规则。
- `conflict_group`：互斥组。
- `formula_hooks`：参与哪些计算。
- `clear_by_*`：可被哪些技能类型清除。
- `special_rule_id`：特殊处理器。

### 5.3 已确认切换规则

| 状态 | category | clear_on_switch | 说明 |
|---|---|---:|---|
| 中毒 | abnormal | true | 可以通过切换清除 |
| 灼烧 | abnormal | true | 可以通过切换清除 |
| 冰冻 | abnormal | false | 不可以通过切换清除，会跟随精灵，除非使用技能去除 |
| 萌化 | special_status | false | 不可以通过切换清除，会跟随精灵，除非使用技能去除 |
| 普通属性增益 | stat_modifier | true | 默认切换清除，除非技能写明永久、继承或队伍侧 |
| 普通属性减益 | stat_modifier | true | 默认切换清除，减少攻击、减少防御等属于此类 |
| 印记 | mark | false | 不因切换清除，除非技能清除、转换或消耗印记 |
| 天气 | weather | false | 战场状态，不跟随精灵切换 |
| 奉献 | special_status | 暂不实现 | 对虫系技能提供额外词条，本阶段仅预留特殊规则 |

切换处理流程：

```text
发生精灵切换
↓
读取离场精灵身上的 BattleEffectInstance
↓
清除 clear_on_switch = true 的实例
↓
保留 clear_on_switch = false 的实例
↓
保留 owner_scope = side 的队伍侧状态
↓
保留 owner_scope = field 的战场状态
↓
记录 EffectChangeEvent(change_type = switch_clear / switch_keep)
```

### 5.4 印记规则

印记不再使用“一个增益印记槽 + 一个减益印记槽”的限制。

原因：部分精灵特性可以使场上存在多个不同印记。因此：

- 印记统一作为 `BattleEffectInstance` 存在。
- `EffectDefinition.category = mark`。
- 同一队伍可以同时拥有多个不同印记。
- 印记正面 / 负面只用于展示和技能筛选，不限制实例数量。
- 同名印记再次获得时按 `stack_rule` 处理。
- 星陨属于印记，不因切换清除。

印记列表通过查询状态实例得到：

```text
active_mark_effects = active_effect_instances.filter(category = mark)
```

### 5.5 天气规则

天气也作为 `BattleEffectInstance` 管理。

推荐规则：

- `category = weather`。
- `owner_scope = field`。
- 天气作用于全战场。
- 天气不因切换清除。
- 默认同一时间只存在一个天气。
- 新天气默认替换旧天气。
- 是否未来支持多天气，由 `conflict_group` 和 `conflict_policy` 控制。

### 5.6 奉献处理边界

奉献目前不进入第一阶段具体规则。

处理原则：

- 可以识别并显示奉献状态。
- 可以记录为 `BattleEffectInstance`。
- 第一阶段不实现其对虫系技能的额外词条。
- 如果计算时存在未实现奉献规则，伤害置信度降低并显示提示。

## 六、属性计算规则

### 6.1 通用公式

生命资质：

$$
\text{生命资质} = 1.00 + \text{种族资质} \times 0.02 + \text{个体资质} \times 0.01
$$

非生命五维资质：

$$
\text{五维资质} = \text{种族资质} \times 0.01 + \text{个体资质} \times 0.005
$$

基础值：

$$
\text{基础值} = 10.00 + \text{种族资质} \times 0.5 + \text{个体资质} \times 0.25
$$

通用属性公式：

$$
\text{属性值} = （\text{基础值} + \text{等级} \times \text{资质值}）\times \text{性格修正} + \text{成长值}
$$

性格修正不作用于成长值。

### 6.2 PVP 化简公式

PVP 中等级固定为 60，成长值固定为：

- 生命成长值 = 100。
- 非生命成长值 = 50。

生命属性：

$$
\text{生命属性} =
\operatorname{round}
\left(
\left[
70 + 1.7 \times \text{种族资质} + 0.85 \times \text{个体资质}
\right]
\times \text{性格修正}
+ 100
\right)
$$

非生命属性：

$$
\text{非生命属性} =
\operatorname{ceil}
\left(
\left[
10 + 1.1 \times \text{种族资质} + 0.55 \times \text{个体资质}
\right]
\times \text{性格修正}
+ 50
\right)
$$

规则：

- 生命属性使用四舍五入。
- 非生命属性使用向上取整。
- 没有个体资质的维度，个体资质按 0 计算。
- 性格可以影响生命属性，也可以影响任意非生命属性。

## 七、伤害、动画多段与连击

### 7.1 伤害展示

所有伤害计算结果同时展示：

- 准确伤害值。
- 生命百分比。

敌方配置未确认时展示区间；配置基本确认时展示单值。

### 7.2 伤害数值与扣血百分比

每次攻击可记录：

- 本次攻击造成的整数伤害。
- 本次攻击扣除了多少百分比生命。

扣血百分比可用于辅助反推最大生命。

如果伤害值和扣血百分比对应关系不一致，则该事件标记为低置信度，等待后续校验。

### 7.3 动画多段伤害

动画多段伤害指：动画和跳字看起来像多段，不同段可能显示不同数值，但最后会显示最终总伤害。

处理规则：

```text
动画多段伤害 = 一次伤害事件
只记录最终显示的总伤害
不要求记录每一段伤害
不把动画段数作为连击数
```

示例：

```text
游戏动画显示：42、43、43
最后显示总伤害：128

系统记录：
damage_display_type = visual_total_damage
damage_value = 128
final_total_damage_value = 128
```

### 7.4 连击

连击与动画多段不同。

连击特点：

```text
每一段伤害相同
最终不会显示总伤害
需要根据单段伤害和连击次数计算总伤害
```

示例：

```text
游戏显示：35、35、35
没有最终总伤害

系统记录：
damage_display_type = combo_repeated_damage
per_hit_damage_value = 35
hit_count = 3
computed_total_damage_value = 105
```

连击推算规则：

- 候选配置过滤优先使用 `per_hit_damage_value`。
- 击杀判断使用 `computed_total_damage_value`。
- 如果血量百分比变化可识别，用总伤害校验最大生命。
- 连击次数可以来自技能固定值、状态修正、手动输入或识别推断。

### 7.5 伤害事件类型

| damage_display_type | 含义 | 记录方式 |
|---|---|---|
| `single_damage` | 单次伤害 | 记录 `damage_value` |
| `visual_total_damage` | 动画多段但最终显示总伤害 | 记录最终总伤害，按一次伤害处理 |
| `combo_repeated_damage` | 连击，每段相同且无总伤害 | 记录单段伤害与连击次数，系统计算总伤害 |
| `special_damage` | 特殊结算伤害 | 由特殊规则处理 |

## 八、敌方配置推算

### 8.1 推算目标

系统推算的是敌方具体配置，而不是种族资质。

推算目标包括：

- 性格。
- 个体资质存在的维度。
- 每个个体资质维度的数值。
- 最终生命。
- 最终物攻。
- 最终物防。
- 最终魔攻。
- 最终魔防。
- 最终速度。
- 技能组。
- 当前状态效果。

### 8.2 我方造成伤害时

我方攻击敌方时，主要反推敌方防御侧信息：

- 敌方生命。
- 敌方物防。
- 敌方魔防。
- 敌方防御相关性格。
- 敌方防御相关个体资质。
- 敌方是否存在防御技能减伤。
- 敌方是否存在印记、异常、天气等公式修正。

流程：

```text
读取我方技能
↓
读取我方面板攻击属性
↓
读取敌方候选面板防御属性
↓
读取技能威力、技能类型、属性克制
↓
读取伤害发生时的 BattleEffectSnapshot
↓
形成 DamageFormulaContext
↓
枚举敌方候选配置
↓
计算理论伤害
↓
与实际伤害和扣血百分比对比
↓
排除不匹配候选
↓
保留匹配候选
↓
更新敌方配置置信度
```

### 8.3 敌方造成伤害时

敌方攻击我方时，主要反推敌方攻击侧信息：

- 敌方物攻。
- 敌方魔攻。
- 敌方攻击相关性格。
- 敌方攻击相关个体资质。
- 敌方可能使用的技能。
- 敌方是否存在增伤、连击、天气、印记、异常等影响。

敌方技能已知时，枚举敌方候选配置并与实际伤害对比。

敌方技能未知时，需要联合枚举：

```text
可能技能 × 候选配置
```

再根据当前快照计算理论伤害，保留可解释实际伤害的组合。

## 九、图像识别与手动输入

### 9.1 自动识别内容

系统需要通过图像识别或 OCR 捕获：

- 准备阶段双方六只精灵。
- 当前对位精灵。
- 精灵头像。
- 技能名称。
- 伤害数值。
- 连击单段伤害。
- 连击次数，如果可以稳定识别。
- 动画多段最终显示的总伤害。
- 敌方扣血百分比。
- 生命百分比或血条变化。
- 状态栏所有状态图标。
- 天气图标。
- 印记图标。
- 技能槽修正图标。
- 能量变化。
- 精灵切换信息。

### 9.2 状态识别策略

所有状态统一识别。

```text
截取状态栏 / 天气栏 / 印记区 / 技能槽图标区
↓
识别所有可见图标
↓
匹配 EffectDefinition.effect_id
↓
生成或更新 BattleEffectInstance
↓
形成 BattleEffectSnapshot
```

识图层只输出事实：

- `effect_id`。
- `owner_scope`。
- `owner_side`。
- `owner_elf_id` 或 `owner_skill_slot_id`。
- `layers`。
- `confidence`。

规则层根据 `EffectDefinition` 字段决定如何参与计算。

### 9.3 手动输入优先级

优先级：

```text
手动输入 > 自动识别 > 系统推算 > 数据库默认值
```

手动输入用于修正精灵、技能、伤害、连击、扣血百分比、状态层数、天气、印记、异常、防御技能状态和准备阶段阵容。

## 十二、当前边界条件与假设

当前版本基于以下假设：

- 等级固定为 60。
- PVP 成长值固定为生命 100，非生命 50。
- 所有 PVP 精灵统一按照 6 星、0 觉醒、50 成长等级计算。
- 敌方六只精灵在准备阶段可见。
- 己方六只精灵完整配置由玩家提前输入。
- 精灵有唯一 ID、名称、头像。
- 精灵没有别名字段。
- 敌方精灵种族资质可由 `elf_id` 从数据库读取。
- 敌方未知配置主要包括性格、个体资质分布、技能组。
- 个体资质一定存在于 1 到 3 个维度。
- 性格必定包含一个正面修正和一个负面修正。
- 速度决定常规出手顺序。
- 同速时，双方各有 50% 概率先出手。
- 所有状态统一进入 `BattleEffectSystem`。
- 状态是否切换清除由 `clear_on_switch` 字段决定。
- 中毒、灼烧可以通过切换清除。
- 冰冻、萌化不能通过切换清除，会跟随精灵，除非使用技能去除。
- 印记是 `BattleEffect` 的一种，不使用唯一增益印记槽和减益印记槽。
- 同一队伍可以同时存在多个不同印记。
- 星陨属于印记。
- 天气是 `BattleEffect` 的一种，作用范围为全战场。
- 动画多段伤害按一次最终总伤害处理。
- 连击按单段伤害和连击次数处理。
- 连击每段伤害相同，最终不显示总伤害。
- 当前游戏没有通用护盾机制。
- 当前阶段暂不考虑命中率。
- 奉献对虫系技能提供额外词条，第一阶段暂不讨论具体实现，仅预留特殊规则字段。

## 十三、仍需确认的问题

### 13.1 属性计算

- 生命属性四舍五入的具体规则是否为标准四舍五入。
- 非生命属性是否始终向上取整。
- 如果后续出现特殊模式，是否仍沿用当前 PVP 属性公式。

### 13.2 伤害公式

- 完整伤害公式是什么。
- 伤害公式每一步是否取整。
- 最终伤害是向下取整、四舍五入还是向上取整。
- 技能威力、攻防属性、克制倍率的计算顺序是什么。
- 普通属性增减是先修正属性，还是作为公式倍率处理。
- 多个增伤、减伤、天气、印记倍率的乘法顺序是什么。
- 防御技能减伤在伤害公式中的生效位置是什么。
- 是否存在伤害上限或下限。
- 连击每段伤害是否完全等于一次普通伤害公式结果。
- 连击总伤害是否始终等于单段伤害 × 连击次数。

### 13.3 状态规则

- 每个印记的最大层数。
- 不同印记之间是否存在个别互斥关系。
- 同名印记再次获得时是叠加、刷新还是取最大值。
- 印记驱散技能是否驱散所有印记，还是只驱散指定方向或指定类型。
- 中毒、灼烧切换清除时是否清除所有层数。
- 萌化最大层数与具体公式影响。
- 冰冻解除条件。
- 各类技能槽修正是否随切换清除。

### 13.4 连击规则

- 所有连击技能的基础连击次数。
- 连击次数被多个状态同时修改时的计算顺序。
- 连击数 +N 和连击数 ×倍率同时存在时的优先级。
- 连击数固定为 2 是否覆盖所有加减和倍率。
- 连击每段伤害是否都可以单独触发附加效果。
- 连击是否每段都消耗某些次数型状态。

## 十五、总结

本系统在洛克王国世界 PVP 对战中，通过准备阶段获取双方六只精灵信息。己方精灵完整配置由玩家提前输入，敌方精灵仅确认种类，具体性格、个体资质分布和技能组未知。

系统根据数据库为敌方每只精灵生成候选配置集合，并在战斗过程中通过实际整数伤害、扣血百分比、技能信息、状态图标、天气、印记、异常、连击、防御技能减伤、增伤 / 减伤等信息，不断过滤和收敛敌方候选配置。

所有状态统一由 `BattleEffectSystem` 管理。状态通过字段区分作用范围、显示分区、生命周期、层数、清除规则、切换是否清除、是否参与公式、是否可被驱散或转换。

伤害计算不依赖持久化的“战斗有效属性”，而是使用面板属性、当前状态快照和伤害公式上下文进行计算。动画多段伤害按最终显示总伤害作为一次伤害处理；连击按单段伤害与连击次数处理。

最终系统实时展示我方技能对敌方的伤害值和生命百分比、敌方已知或可能技能对我方的伤害值和生命百分比、连击总伤害、双方速度关系与先手概率，为玩家提供可靠的战斗数据支持。
